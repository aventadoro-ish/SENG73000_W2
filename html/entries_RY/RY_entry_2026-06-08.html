Professor's Advice

  • No need for the position sensor since the SPI sensor is precise
  • Submit the video of STM32 CAN communication
  • Wants to see traffic of messages; all controllers should see the message from one sent message (serial monitor)
  • Think about cart request, floor request, and remote request of elevator movement — consider priority ordering
  • Each level should have a queue for when multiple buttons are pressed
  • Need to track if someone is inside the elevator; disable doors when someone is inside or when moving
  • Matvey: simulate someone inside using a toggle switch inside the elevator — should be part of the code
  • Matvey: for future phases, additional features include door opening and sensing for someone inside
  • Safety: timeout protection — if the controller does not receive a message within 10 seconds, trigger a fault condition and stop the elevator. Multiply timeout by 2 for travel between floors 1 and 3
  • E-stop should be connected to the Cart Controller — software only, no dedicated hardware E-stop required
  • Motor selection: NEMA 17, 23, and 34 — select the correct current for the motor driver
  • Some drivers support microsteps; using bipolar configuration could allow for very precise positioning
